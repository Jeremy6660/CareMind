## 整体思路说明

这个方案的核心逻辑是：**VS Code** → **Cline（临时智能体插件）** → **Claude Code**。

Cline 本质上是 VS Code 里的一个"AI 助手"，它可以听懂人类的指令，直接在终端里执行命令、操作文件。我们利用 Cline 的这个能力，让它代劳安装 Claude Code，电脑小白只需要操作到"装好 Cline 并完成配置"这一步，之后的事由 Cline 自动完成。

整个流程分为四个阶段：

| 阶段 | 操作内容 | 说明 |
|------|---------|------|
| 第一阶段 | 确认系统环境 | 检查 Node.js 和 Git 是否已安装 |
| 第二阶段 | 安装 VS Code | 官方下载，注意 PATH 选项 |
| 第三步 | 安装 Cline 插件 | 在 VS Code 扩展商店搜索安装 |
| 第四步 | 配置 Cline | 填入 API Key，选择模型 |
| 第五步 | 用 Cline 安装 Claude Code | 输入一句话指令，Cline 自动执行 |
| 第六步 | 验证与收尾 | 确认安装成功，测试使用 |


## 第一阶段：准备工作——确认系统环境

在正式安装之前，需要先确认两样东西。这两样是 Claude Code 的依赖，**缺一不可**：

### 1. 确认 Node.js 版本 ≥ 18

**为什么需要 Node.js？** Claude Code 基于 Node.js 运行，这是它最核心的运行时环境。

**怎么检查？**
- 按键盘上的 `Win + R`，输入 `cmd`，回车
- 在黑色窗口里输入 `node --version`，回车
- 如果显示类似 `v18.x.x` 或 `v20.x.x` 或 `v22.x.x`，说明已安装且版本达标
- 如果显示 `v16.x.x` 或更低，说明版本不够，**必须升级到 18 以上**
- 如果显示 `'node' 不是内部或外部命令`，说明**完全没装 Node.js**

> ⚠️ **这是一个大坑**：大多数安装失败的原因都是 Node.js 版本低于 18。这一步必须确认清楚，否则 Claude Code 装不上。

**如果没装或版本不够怎么办？**
1. 访问 Node.js 官网 **nodejs.org**
2. 下载 **LTS 版本**（长期支持版，推荐 v22.x）
3. 双击安装，一路点"下一步"，全部默认选项就行
4. 装完后**重新打开 cmd**，再次运行 `node --version` 确认


### 2. 确认 Git 是否安装

**为什么需要 Git？** Windows 上 Claude Code 底层依赖 Git Bash 来执行 shell 命令。

**怎么检查？**
- 同样在 cmd 窗口中输入 `git --version`，回车
- 如果显示版本号（如 `git version 2.47.x`），说明已安装
- 如果显示 `'git' 不是内部或外部命令`，说明没装

> ⚠️ **第二个常见坑**：Windows 上缺少 Git 会导致 Claude Code 安装后无法正常工作。

**如果没装怎么办？**
1. 访问 **git-scm.com**
2. 下载 Windows 版本
3. 安装时一直点"下一步"，默认选项即可
4. 装完后**重新打开 cmd**，再次输入 `git --version` 验证


## 第二阶段：安装 VS Code

### 下载和安装步骤

1. 打开浏览器，访问 **code.visualstudio.com**（这是唯一官方下载地址，不要从其他地方下载）
2. 页面会自动识别你的系统，点击大大的 **Download** 按钮
3. 下载完成后，双击安装文件（文件名类似 `VSCodeUserSetup-x64-xxx.exe`）

### ⚠️ 安装过程中的关键一步

安装向导走到"选择附加任务"这个页面时，有一个选项**非常关键**：

> ✅ **一定要勾选"添加到 PATH"这个选项！**

**为什么要勾选？** 勾选后可以在终端的任意位置输入 `code` 命令启动 VS Code，后续 Cline 也需要用到这个能力。不勾选的话，终端里输入 `code` 会提示找不到命令。

其他选项保持默认即可，一路点"下一步"直到安装完成。

### 可选但强烈推荐：安装中文语言包

VS Code 默认是英文界面。如果觉得英文不习惯：

1. 打开 VS Code
2. 点击左侧竖条上的 **扩展图标**（四个小方块组成的图标），或者按快捷键 `Ctrl + Shift + X`
3. 在搜索框里输入 **Chinese**
4. 找到微软官方发布的 **"Chinese (Simplified) (简体中文) Language Pack"**，点击 **Install（安装）**
5. 安装完成后，VS Code 右下角会弹出提示，点击 **"Change Language and Restart"** 即可切换为中文


## 第三阶段：安装 Cline 插件

### 安装步骤

1. 打开 VS Code
2. 点击左侧活动栏的 **扩展图标**（四个小方块），或按快捷键 `Ctrl + Shift + X`
3. 在搜索框中输入 **Cline**
4. 找到由 **"saoudrizwan"** 发布的那个 Cline 扩展，确认下载量超过百万的那个（这是唯一正版）
5. 点击右侧蓝色的 **Install（安装）** 按钮
6. 等待安装完成，左侧活动栏会出现一个 Cline 的图标

> ⚠️ **注意区分**：扩展商店里可能有类似名字的插件，一定要认准 **saoudrizwan** 发布的原版 Cline，下载量最大的那个。


## 第四阶段：配置 Cline（让 AI 能工作）

Cline 是一个 AI 智能体，它需要连接到一个大语言模型才能运行。也就是说，**Cline 本身只是一个"框架"，背后必须接入 AI 模型才能工作**。

### 最关键的准备工作：获取 API Key

Cline 需要你的 API Key 才能调用 AI 模型。你有以下选择：

**方案一：使用 Anthropic 官方 API Key（推荐给有条件的用户）**
1. 访问 **console.anthropic.com**，注册/登录开发者账号
2. 进入 **API Keys** 页面
3. 点击 **Create Key**，复制生成的密钥（格式为 `sk-ant-xxx...`）

> ⚠️ **特别注意**：API Key 只显示一次，生成后**立即复制保存**，关掉页面就再也看不到了！

**方案二：使用中转 API 服务（国内用户推荐）**
如果无法访问 Anthropic 官网或没有国外信用卡，可以使用国内云厂商提供的中转服务，例如阿里云百炼的 Coding Plan，接入后可正常使用。

> 这个环节是**整个流程最大的坑之一**。Cline 必须要有 API Key 才能工作，没有 API Key = Cline 完全不能用 = 没办法让它帮忙装 Claude Code。安装前务必确认 API Key 已准备好。


### 在 Cline 中配置 API

1. 打开 VS Code，点击左侧活动栏的 **Cline 图标**
2. 如果是首次打开，会看到几个选项，选择 **"Bring my own API key"**（使用自己的 API Key）
3. 在弹出的配置页面中：
   - **API Provider**（API 提供商）：选择 **Anthropic**（如果使用中转服务，选择 **OpenAI Compatible** 并填入对应的 Base URL）
   - **API Key**：粘贴你获取到的 API Key
   - **Model**（模型）：选择 **claude-sonnet-4-5** 或类似可用模型
4. 配置完成后点击 **Done** 或 **Continue**

### ⚠️ 关于 Cline 免费额度的提醒

Cline 新用户可能会有一定的免费额度，但通常有限。如果额度用完，调用 API 就会失败。不过我们只是用 Cline 来安装 Claude Code 这一个临时任务，通常消耗的 Token 非常少，免费额度大概率够用。

> **额外的坑**：如果你的网络环境无法直接访问 Anthropic 的 API（api.anthropic.com），则需要在 Cline 中配置中转 Base URL，否则 API 调用会失败。


## 第五阶段：用 Cline 安装 Claude Code

这是最"魔法"的一步——让 Cline 这个 AI 助手帮你完成 Claude Code 的安装。

### 操作步骤

1. 打开 VS Code，点击左侧的 **Cline 图标**
2. 在 Cline 的对话框中，输入以下指令：

```
请帮我在当前电脑上安装 Claude Code。
Windows 系统推荐使用 PowerShell 原生安装方式：
irm https://claude.ai/install.ps1 | iex

安装完成后，请把 %USERPROFILE%\.local\bin 添加到系统环境变量 PATH 中，然后验证 claude --version 是否能正常运行。

如果需要管理员权限，请提前告知我。
```

3. 按回车发送

4. 接下来 Cline 会自动执行：
   - 打开终端
   - 运行安装命令
   - 配置 PATH
   - 验证安装结果

5. 过程中如果 Cline 弹出权限确认提示（比如"Allow"或"Run"），点击 **Allow（允许）** 即可

### ⚠️ 这一步可能遇到的问题

**问题 1：PowerShell 执行策略限制**

如果 Cline 报错说 PowerShell 无法运行脚本，这是因为 Windows 默认禁止运行外部脚本。解决方案：

在 Cline 中输入以下指令让 Cline 先解除限制：
```
请先运行这条命令解除 PowerShell 脚本执行限制：
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
然后再安装 Claude Code。
```


**问题 2：PATH 配置**

安装完成后，如果提示 `'claude' 不是内部或外部命令`，说明 PATH 没有配好。让 Cline 帮忙修复：

```
请把 %USERPROFILE%\.local\bin 添加到系统环境变量 PATH 中
```

**问题 3：Windows 原生安装 vs npm 安装**

2026 年 Anthropic 推荐使用**原生安装方式**（PowerShell 一键脚本），弃用了旧的 npm 安装方式。直接用上面的 PowerShell 命令安装即可。

Windows 上安装完 Claude Code 后，还需要确保 Git 已安装（第一步已确认过），因为 Claude Code 在 Windows 上需要用 Git Bash 执行命令。


## 第六阶段：验证与收尾

安装完成后，打开一个新的终端窗口（cmd 或 PowerShell），输入：

```
claude --version
```

如果显示了版本号（如 `v1.x.x`），说明安装成功！

如果显示 `'claude' 不是内部或外部命令`，说明 PATH 还没生效，可以：
1. **关闭当前终端，重新打开一个新的终端**再试
2. 或者重启电脑（这是最彻底的方法）


## ⚠️ 整个流程中最大的几个"坑"总结

| 序号 | 坑 | 原因 | 解决方案 |
|------|-----|------|---------|
| 1 | **Node.js 版本不够** | 低于 18 会导致安装失败 | 升级到 Node.js 18+，推荐 22 LTS |
| 2 | **Git 没装** | Windows 上 Claude Code 依赖 Git Bash | 从 git-scm.com 下载安装 |
| 3 | **VS Code 没加 PATH** | 终端找不到 `code` 命令 | 安装时勾选"添加到 PATH" |
| 4 | **API Key 没有或无效** | Cline 需要 API Key 才能工作 | 提前准备好有效的 API Key |
| 5 | **API Key 只显示一次** | 生成后不保存就永远丢失 | 生成后立即复制粘贴到安全的地方 |
| 6 | **PowerShell 执行策略** | Windows 默认禁止运行远程脚本 | 先运行 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` |
| 7 | **安装后找不到 claude 命令** | PATH 没配或没生效 | 添加 `%USERPROFILE%\.local\bin` 到 PATH，然后重启终端 |
| 8 | **网络无法访问 Anthropic 域名** | 国内网络限制 | 配置中转 API 服务或使用代理 |
| 9 | **Cline 自动执行时被安全软件拦截** | 杀毒软件误报 | 暂时关闭杀毒软件，或在提示时选择"允许" |

> 📌 **补充提醒**：Cline 在执行安装任务时会自动操作终端和文件系统，整个过程对小白来说可能看起来有些"吓人"（终端窗口突然弹出、代码疯狂滚动）。这是正常的，不需要人工干预，等 Cline 自己跑完就行。但如果超过 **5 分钟**还没完成，可以在 Cline 对话框中输入"请继续"来推动它。
