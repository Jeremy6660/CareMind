# 系统设计文档：CareMind 多智能体照护系统

> **完整的系统架构、Agent设计、个性化机制、技术栈与执行计划**

---

## 一、项目背景与问题定位

### 核心痛点

中国老龄化进程持续加速，失能与半失能老人照护面临三重结构性困难：

- **信息孤岛**：老人病历、用药记录、日常状态分散在不同系统，照护人员难以整合
- **沟通断层**：专业医疗信息与家属认知存在巨大鸿沟，家属长期处于"不知情"状态
- **个性缺失**：现有照护系统以机构管理为导向，采用打勾式标准化记录，忽视老人作为个体的差异化需求

### 研究切口

本项目聚焦于**养老院与居家照护场景**中，照护人员与家属之间的信息不对称、用药安全风险、以及照护沟通的个性化问题，提出一套以老人为中心的多智能体协同辅助系统解决方案。

---

## 二、系统定位与创新点

### 系统名称

**CareMind**：面向失能老人照护的多智能体协同个性化决策系统

### 三大创新点

| 创新维度 | 具体内容 |
|---------|---------|
| **个性化生成** | 根据认知能力、方言偏好、病史动态生成差异化照护内容 |
| **多智能体协同** | 三个专职Agent分工协同，通过Orchestrator统一调度决策 |
| **以人为中心的叙述框架** | 突破管理主义文档范式，以老人主体性为核心重构照护记录 |

## 四、三大Agent详细设计

### Agent 1：分诊评估

**核心职责**：判断老人当前健康状态，评估风险等级，决定是否触发其他Agent介入

**输入**：
- 老人当日口述或照护人员录入的症状描述
- 历史健康记录与基线数据

**输出**：
- 风险等级（绿 / 黄 / 红）
- 差异化处置建议
- 是否触发 Agent 2 / Agent 3

**个性化机制**：

| 认知状态 | 交互方式 |
|---------|---------|
| 认知正常 | 直接对话，主动询问症状细节 |
| 轻度认知障碍 | 简化语言，图片辅助，是/否式选项 |
| 中重度认知障碍 | 转由照护人员代理输入，系统辅助分析 |

---

### Agent 2：用药安全

**核心职责**：核查多药并用风险，生成个性化用药提醒与预警

**背景**：中国老年人平均同时服用5种以上慢性病药物，药物相互作用是重大安全隐患，现有照护系统对此几乎无覆盖。

**输出示例**：

```
⚠️ 用药风险预警

检测对象：张奶奶（77岁）
检测到：阿司匹林 + 布洛芬 同时服用
风险评估：增加消化道出血风险（证据等级 B）

建议措施：
→ 提示照护人员告知主治医生
→ 考虑以对乙酰氨基酚替代布洛芬
→ 已生成家属通知草稿 [点击查看]
```

---

### Agent 3：家属沟通

**核心职责**：将专业照护信息转化为家属可理解的语言，定期生成人性化照护报告

**设计哲学**：突破管理主义的"打勾式文档"范式，以老人的生命主体性为叙述核心，还原其作为"人"而非"患者"的真实状态。

**输出示例**：

```
【本周李爷爷状态报告】

今天他主动聊起了年轻时在工厂的故事，情绪很好。
下午散步时走了20分钟，比上周多了5分钟。

用药情况：按时服药，血压平稳（135/82）。

需要您关注的事：
他最近提到很想吃家里做的红烧肉，
如果方便下次探视时可以带一份 ❤️

下周照护计划：尝试引导参与园艺小组活动。
```

> 对比冷冰冰的机构报告，这种输出风格直接触达家属情感，也更真实地呈现了老人的需求与状态。

## 五、个性化生成机制

"个性化"不是噱头，需要有具体的技术实现路径：

| 个性化维度 | 实现方式 |
|----------|---------|
| **语言复杂度** | 根据认知评级动态调整输出字数、句子长度、词汇难度 |
| **方言适配** | TTS方言模型辅助语音播报，文字注音辅助 |
| **关系视角** | 同一照护信息针对老人 / 护工 / 家属生成三个版本 |
| **时序个性化** | 记忆模块持续记录老人偏好与变化，动态微调输出（RAG + 记忆更新） |

## 六、技术栈方案

| 模块 | 推荐技术 |
|------|---------|
| 多Agent框架 | LangGraph / AutoGen |
| 大语言模型 | GPT-4o / Qwen2.5（可部署本地） |
| 知识库检索 | RAG（LlamaIndex + 向量数据库） |
| 用户画像存储 | MongoDB |
| 前端交互 | 微信小程序 / Web端 |
| 语音方言支持 | 阿里云语音TTS方言模型 |

|---------|----------|
