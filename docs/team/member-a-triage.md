# 成员 A 分工安排：Agent 1 分诊评估负责人

## 一、团队总分工

CareMind 采用“四条主线、互相验收”的小组协作方式。每个人负责一个清晰模块，同时承担对其他成员产出的审查责任，避免最后集成时才发现接口、内容或安全边界不一致。

| 成员 | 角色定位 | 主责模块 | 核心交付物 | 评价指标 |
|---|---|---|---|---|
| A | 分诊评估负责人 | Agent 1 | 症状案例库、分诊 Prompt、分诊函数、测试记录 | 风险分级准确率、建议是否安全清晰 |
| B | 用药安全负责人 | Agent 2 | 药物风险 JSON、用药匹配逻辑、预警 Prompt、来源审核表 | 高风险组合不漏报、来源可追溯 |
| C | 家属沟通负责人 | Agent 3 | 老人画像、报告模板、沟通 Prompt、三方报告生成 | 表达温暖、信息准确、不掩盖风险 |
| D | 集成与交付负责人 | Orchestrator + Demo + 材料 | 项目骨架、Gradio Demo、端到端流程、报告/PPT 汇总 | Demo 可演示、接口统一、材料完整 |

互审机制：

- A 审 C：确认家属报告里的健康风险没有被“温暖表达”稀释。
- B 审 A：确认分诊建议没有越权诊断或医学风险。
- C 审 B：确认用药预警能被护工和家属读懂。
- D 审全员：确认接口、文件结构、演示流程可以集成。

## 二、A 的角色定位

A 是 Agent 1 分诊评估负责人，负责把护工或老人输入的症状描述转化为清晰、保守、安全的风险判断。

Agent 1 是整个系统的入口，决定是否需要触发 Agent 2 用药安全检查，是否需要提醒 Agent 3 在家属报告中加入重点关注事项。因此 A 的产出必须做到结构稳定、风险边界清楚、便于 D 接入 Orchestrator。

## 三、阶段任务

| 阶段 | 时间 | 任务重点 | 产出 |
|---|---|---|---|
| 准备期 | 6 月 | 准备症状案例库与分诊 Prompt | `test_cases_agent1.json`、Agent 1 Prompt 初版 |
| 开发期 | 7 月 | 将 Prompt 封装为 Python 函数 | `triage_assessment()`、测试记录 |
| 集成期 | 8 月 | 参与端到端测试，修正风险等级和触发逻辑 | 5 个完整案例中的分诊结果 |
| 冲刺期 | 9 月 | 汇总测试结果，支持报告和答辩材料 | 测试准确率、典型案例、相关工作材料 |

## 四、具体措施

1. 建立不少于 10 个老人症状测试案例，建议扩展到 20 个。
2. 案例必须覆盖绿色、黄色、红色三类风险，其中绿色不少于 3 个、黄色不少于 4 个、红色不少于 3 个。
3. 案例必须覆盖三种认知状态：认知正常、轻度认知障碍、中重度认知障碍。
4. 每个案例至少包含：老人年龄、认知状态、症状描述、既往病史、预期风险等级、预期处置建议。
5. 设计分诊 Prompt，输出固定包含：风险等级、判断理由、处置建议、是否触发 Agent 2、是否触发 Agent 3。
6. 开发函数 `triage_assessment(symptoms, history, cognitive_level) -> dict`。
7. 每次修改 Prompt 后，用全部测试案例重新跑一遍并记录准确率。

建议输出结构：

```json
{
  "risk_level": "yellow",
  "reason": "老人出现头晕、进食减少，且有高血压病史，需要持续观察。",
  "action": "建议测量血压并观察 2 小时，如症状持续或加重，联系医生复核。",
  "trigger_medication_agent": true,
  "trigger_family_report": true,
  "safety_note": "本结果仅作为照护辅助建议，不替代医生诊断。"
}
```

## 五、工作要求

- 不能输出“确诊”“必须用药”“必须停药”“系统诊断为”等医疗越权表述。
- 红色风险必须建议及时联系医生、护士或急救资源。
- 黄色风险要给出可执行的观察建议，例如测血压、观察时长、记录症状变化。
- 绿色风险也要给出日常照护建议，不能只输出“无风险”。
- 面向轻度认知障碍老人时，语言要更短、更直接。
- 面向中重度认知障碍老人时，应提示由护工或家属代理输入信息。
- 输出字段必须稳定，方便 D 在 Orchestrator 中调用。

## 六、验收标准

- 6 月底：完成不少于 10 个症状案例和 Agent 1 Prompt 初版，Prompt 初测准确率不低于 70%。
- 7 月底：`triage_assessment()` 可以独立运行，风险分级准确率不低于 80%。
- 8 月底：在 5 个端到端案例中，分诊结果能正确触发 Agent 2 或 Agent 3。
- 9 月：提供 1-2 个适合答辩展示的典型分诊案例，并整理测试结果说明。

## 七、注意事项

- Agent 1 的价值不是“替医生诊断”，而是帮助护工判断是否需要进一步关注。
- 对不确定情况要保守处理，宁可建议复核，也不要给出过度肯定的判断。
- 分诊建议应符合一线护工能理解、能执行的表达方式。
- 测试案例要尽量贴近养老院真实场景，避免只写教科书式症状。
